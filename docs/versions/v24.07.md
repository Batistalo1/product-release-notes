##  New Features

- It is now possible to factory reset a controller.
- A new parameter has been added to ignore UID tags when OSS/EID is set.
- Office function can now be assigned to single permissions.
- The global performance of event collection has been improved.
- The ACM default display LED is now BLUE instead of red (ACM Firmware 5.9).
- The Redis file is now added to the backup of the controller so that offline devices are no longer missing after a restore.
- A trace level has been added for the API logs.

---

##  Fixes

- It is now impossible to remove a device from a RFNM using the route-devices API route.
- Devices will no longer be out-of-sync when a configuration change is sent but does not impact them directly.
- Some requests are no longer bound to device state. If a device state is not correct to pull a request, the request will be stored and pushed when the state is ready.
- The online configuration of RFNM/ACM is now up to date after changing IP settings.
- The stability of coupling using the DOMService app has been improved.

---

##  Known Issues

Synchronization with the DOMService app is sometimes stuck.

**Workaround:** A workaround is implemented in DOMPloy 12.0 by clicking on the **Offline** tab.
